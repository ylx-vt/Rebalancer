# 实时估值再平衡观测开发设计文档

本文面向 `实时估值再平衡观测` 开发评审，说明功能落地时的模块边界、数据结构、接口验证、计算流程、界面集成、异常降级和测试验收方案。

对应需求文档：[实时估值再平衡观测 PRD](../需求文档/实时估值再平衡观测PRD.md)

## 1. 开发目标

在不改变现有确认净值观测和历史回测口径的前提下，为观测页增加一个可切换的实时估值口径。

本期交付目标：

1. 观测页支持 `确认净值` 和 `实时估值` 两种价格口径切换。
2. 默认仍使用确认净值，切换状态只影响当前观测展示，不修改配置。
3. 实时估值模式优先使用东财批量估值接口返回的估算净值。
4. 当接口表明当日真实净值已更新时，实时估值模式自动改用当日 `NAV`。
5. 单只基金估值缺失时回退确认净值，单只请求失败不影响其他基金。
6. 页面展示每只基金的价格来源、估值时间、组合级差异和估值触发标签。
7. 历史回测继续只使用确认净值，不依赖实时估值模块。

## 2. 当前工程基础

现有工程已经具备以下能力：

1. `src/domain/types.ts` 定义配置、持仓、基金快照、历史净值和观测结果。
2. `src/domain/calculation.ts` 负责确认净值口径下的当前观测计算。
3. `src/services/fundApi.ts` 已封装单只基金快照、历史净值和历史净值序列请求。
4. `src/services/fundClient.ts` 已实现基金数据缓存、批量加载和单只失败隔离。
5. `src/background.ts` 作为 MV3 service worker 代理跨域基金数据请求。
6. `src/services/storage.ts` 使用版本化 `AppState` 持久化配置、缓存和最近一次回测参数。
7. `src/ui/App.tsx` 承载 Popup 内观测、配置、回测等主流程。

实时估值应作为当前观测能力的扩展，不应写入回测引擎，也不应让 `calculatePortfolioObservation` 同时承担多种价格来源解释。建议新增独立 quote 类型和估值选择层，再复用或轻量泛化现有观测计算。

## 3. 接口可用性验证

### 3.1 验证结论

已按 PRD 参数直接请求东财批量估值接口：

```text
GET https://fundmobapi.eastmoney.com/FundMNewApi/FundMNFInfo
```

验证时间：`2026-04-27 14:46` 北京时间。

验证样例：

```text
Fcodes=000001,110022,161725,003096,005827,001186
```

接口返回：

1. HTTP 状态为 `200`。
2. 响应头包含 `Access-Control-Allow-Origin: *`，浏览器跨域访问条件成立。
3. 响应体包含 `ErrCode: 0`、`Success: true`、`TotalCount` 和 `Datas`。
4. `FCODE`、`SHORTNAME`、`PDATE`、`NAV`、`NAVCHGRT` 能正常返回。
5. 当前验证样例中 `GSZ`、`GSZZL`、`GZTIME` 均为 `null`。
6. `Expansion` 返回了全局日期信息，例如 `GZTIME: "2026-04-27"`、`FSRQ: "2026-04-24"`。

### 3.2 设计影响

接口本身可访问，但实时估值字段当前未返回有效值。因此开发设计必须支持两层判断：

1. 接口成功不等于实时估值可用。
2. 单只基金 `GSZ` 为空时必须回退到确认净值，并展示 `暂无估值` 或 `净值`。

实施前需要再次在真实交易时段验证是否存在 `GSZ` 非空样例。如果连续验证仍为空，本功能的核心价值会被降级为“批量确认净值观测”，需要由产品侧决定是否继续使用该接口或调整数据源。本设计不自行引入 PRD 范围外的多 provider。

### 3.3 Manifest 权限

当前 `public/manifest.json` 还未声明新接口域名，需要新增：

```json
"https://fundmobapi.eastmoney.com/*"
```

当前响应头允许跨域，但 Chrome 扩展仍应显式声明 host permission，保持 MV3 请求路径稳定。

4. UI 只处理模式切换、加载状态和结果展示。
5. 回测模块不引入实时估值类型。

## 6. 接口与解析设计

### 6.1 底层接口

在 `src/services/fundApi.ts` 新增：

```ts
export const fetchFundQuotes = async (
  codes: string[],
  deviceId: string
): Promise<FundQuote[]>
```

请求规则：

1. 去重并过滤空代码。
2. 每批最多 200 只基金。
3. `Fcodes` 使用逗号分隔。
4. 继续使用 `withTimeout`，超时仍为 10 秒。
5. 返回 `ErrCode !== 0` 或 `Datas` 非数组时抛出可读错误。
6. 单条记录字段异常不抛出整个批次，记录为该基金报价不可用。

请求参数：

```text
pageIndex=1
pageSize=200
plat=Android
appType=ttjj
product=EFund
Version=1
deviceid={valuationDeviceId}
Fcodes={codes.join(",")}
```

### 6.2 字段标准化

解析函数保持纯函数，便于单测：

```ts
export const normalizeFundQuote = (raw: Record<string, unknown>): FundQuote
```

规则：

1. `FCODE` 转为 `code`，为空则丢弃该行。
2. `SHORTNAME` 转为 `name`，为空时保留配置名称。
3. `NAV` 可转数字则为 `confirmedNav`，否则为 `null`。
4. `GSZ` 可转数字则为 `estimatedNav`，否则为 `null`。
5. `GSZZL` 可转数字则为 `estimatedChangeRate`，否则为 `0`。
6. `NAVCHGRT` 可转数字则为 `actualChangeRate`，否则为 `0`。
7. `PDATE === "--"` 时 `confirmedDate` 记为空字符串。
8. `GZTIME` 为空或非字符串时 `estimatedAt` 记为空字符串。
9. `hasActualNavToday` 只由 `PDATE` 与 `GZTIME` 日期部分判断，不用本地时间推断。

### 6.3 当日真实净值判断

```ts
const hasActualNavToday =
  Boolean(confirmedDate) &&
  Boolean(estimatedAt) &&
  confirmedDate === estimatedAt.slice(0, 10);
```

注意：本地日期、周末和交易时段只用于 UI 提示，不参与当日真实净值替代判断。

## 7. 估值加载与缓存

新增 `src/services/valuationClient.ts`。

```ts
export interface LoadFundQuotesResult {
  quotes: Record<string, FundQuote>;
  fundQuoteCachePatch: Record<string, FundQuote>;
  valuationError?: string;
}

export const loadFundQuotesForConfig = async (
  config: PortfolioConfig,
  state: AppState,
  forceRefresh: boolean
): Promise<LoadFundQuotesResult>
```

缓存规则：

1. `fundQuoteCache` 按基金代码存储。
2. TTL 为 60 秒。
3. `forceRefresh=true` 时绕过缓存。
4. 批量接口整体失败时返回 `valuationError`，调用方回退确认净值观测。
5. 单只基金无估值字段但有确认净值时，不算请求失败，按价格来源 `confirmed` 处理。
6. 单只基金 `NAV` 和 `GSZ` 都不可用时，按价格来源 `unavailable` 处理。

`valuationDeviceId` 生成规则：

1. 首次进入实时估值模式时生成 `crypto.randomUUID()`。
2. 不支持 `crypto.randomUUID` 时使用时间戳和随机数组合。
3. 生成后持久化，不随刷新变化。

## 8. 计算流程

### 8.1 共享计算内核

建议把当前 `calculatePortfolioObservation` 拆成两层：

```ts
calculatePortfolioObservation(config, quotes)
calculatePortfolioObservationFromPrices(config, holdingPrices)
```

其中 `calculatePortfolioObservationFromPrices` 接受已经选好的起始价格和当前价格：

```ts
interface HoldingPriceInput {
  holding: HoldingConfig;
  initialNav: number;
  currentNav: number;
  currentNavDate?: string;
  error?: string;
}
```

确认净值模式继续由 `loadQuotesForConfig` 生成 `HoldingPriceInput`。

实时估值模式流程：

1. 用现有 `loadQuotesForConfig` 加载起始历史净值和确认净值快照。
2. 用 `loadFundQuotesForConfig` 加载批量估值报价。
3. 对每只基金调用 `selectEffectivePrice` 选择有效价格。
4. 生成实时估值口径 `HoldingPriceInput`。
5. 同时生成确认净值口径 `HoldingPriceInput`。
6. 分别计算两份 `PortfolioObservation`。
7. 基于两份结果计算组合级差异和持仓级触发差异。

这样可以保证两种口径使用同一套偏离、阈值和调仓公式。

### 8.2 有效价格选择

```ts
export const selectEffectivePrice = (quote: FundQuote): EffectivePrice | null => {
  if (quote.hasActualNavToday && quote.confirmedNav !== null) {
    return {
      nav: quote.confirmedNav,
      source: "actual-today",
      dateLabel: quote.confirmedDate,
      changeRate: quote.actualChangeRate
    };
  }

  if (quote.estimatedNav !== null) {
    return {
      nav: quote.estimatedNav,
      source: "estimated",
      dateLabel: quote.estimatedAt,
      changeRate: quote.estimatedChangeRate
    };
  }

  if (quote.confirmedNav !== null) {
    return {
      nav: quote.confirmedNav,
      source: "confirmed",
      dateLabel: quote.confirmedDate,
      changeRate: quote.actualChangeRate
    };
  }

  return null;
};
```

现金持仓不请求估值接口，价格固定为 `1`，价格来源可视为 `confirmed`。

### 8.3 触发差异

持仓级标签根据确认净值口径和实时估值口径的 `rebalanceStatus` 比较：

```ts
const isTriggered = (status: RebalanceStatus) => status === "rebalance";
```

| 确认净值口径 | 实时估值口径 | `triggerDiff` | 展示 |
| --- | --- | --- | --- |
| 未触发 | 触发 | `estimated-triggered` | `估值触发` |
| 触发 | 触发 | `both-triggered` | `需再平衡` |
| 触发 | 未触发 | `estimated-cleared` | `估值回落` |
| 未触发 | 未触发 | `none` | 不展示额外标签 |

`watch` 只作为观察态，不计入“触发”。

### 8.4 组合级差异

实时估值模式下，额外计算：

```text
组合市值差异 = 实时估值口径总市值 - 确认净值口径总市值
组合偏离差异 = 实时估值口径组合偏离 - 确认净值口径组合偏离
```

组合总览展示一行 diff 文案：

```text
较确认净值口径 +482.20，偏离变化 +0.32pp
```

当 `estimatedCount === 0` 且全部基金回退确认净值时，展示：

```text
暂无可用实时估值，已使用确认净值口径
```

## 9. 时段提示设计

新增轻量函数：

```ts
export type ValuationSessionLabel =
  | "盘中估值"
  | "午间估值"
  | "收盘估值，待净值更新"
  | "当日净值已更新"
  | "非交易日，使用确认净值"
  | "暂无估值";
```

判断规则：

1. 周六、周日直接视为非交易日。
2. 周一至周五按北京时间判断交易时段。
3. 当 `actualTodayCount > 0` 且无 `estimated` 来源时，优先展示 `当日净值已更新`。
4. 当 `estimatedCount === 0` 时展示 `暂无估值` 或 `非交易日，使用确认净值`。
5. 法定节假日不在本期维护。

该标签仅用于解释页面状态，不参与价格选择。

### 10.2 刷新流程

确认净值模式：

1. 调用现有 `loadQuotesForConfig`。
2. 计算确认净值观测。
3. 更新 `fundCache` 和 `historicalCache`。

实时估值模式：

1. 并行加载确认净值 quote、估值 quote、基准数据。
2. 确认净值 quote 失败时按现有错误处理。
3. 估值 quote 整体失败时展示提示，并回退确认净值观测。
4. 估值 quote 成功但单只估值为空时逐只降级。
5. 更新 `fundQuoteCache`、`fundCache`、`historicalCache`。

切换到实时估值模式时：

1. 若缓存 60 秒内有效，立即重算。
2. 若缓存缺失或过期，触发估值加载。
3. 不修改当前配置。

### 10.3 展示调整

观测页顶部增加分段控件：

```text
确认净值 | 实时估值
```

实时估值总览展示：

1. 组合估算总市值。
2. 与确认净值口径的市值差异。
3. 与确认净值口径的组合偏离差异。
4. 估值触发数量。
5. 最新估值时间或降级提示。
6. `8/10 只使用实时估值，2 只沿用确认净值` 这类覆盖说明。

持仓明细增加：

1. 价格来源标签：`估值`、`当日净值`、`净值`、`暂无估值`。
2. 有效价格和价格日期。
3. 价格相对确认净值的差异率。
4. `估值触发`、`估值回落` 标签。

视觉上不新增行情图，不使用高饱和涨跌色，确保 Popup 信息密度可控。

## 13. 异常与降级

| 场景 | 处理 | UI 提示 |
| --- | --- | --- |
| 批量估值接口 HTTP 失败 | 整体回退确认净值观测 | `估值获取失败，已使用确认净值` |
| 批量估值接口业务失败 | 整体回退确认净值观测 | `估值返回异常，已使用确认净值` |
| 单只基金无 `GSZ` | 使用 `NAV` | `暂无估值` 或 `净值` |
| 单只基金 `NAV` 和 `GSZ` 都不可用 | 该持仓错误，其他持仓继续 | `价格不可用` |
| `GZTIME` 为空 | 不展示估值时间 | `估值时间未知` |
| 周末打开实时估值模式 | 默认回退确认净值 | `非交易日，使用确认净值` |
| `PDATE == GZTIME` 日期部分 | 使用 `NAV` 替代 `GSZ` | `当日净值已更新` |

## 14. 测试计划

### 14.1 单元测试

重点覆盖：

1. `normalizeFundQuote` 对数字、空值、`--`、`null` 的标准化。
2. `hasActualNavToday` 只依赖 `PDATE` 和 `GZTIME` 日期部分。
3. `selectEffectivePrice` 的四种来源选择。
4. `calculateEstimatedObservation` 与确认净值计算公式一致。
5. `HoldingTriggerDiff` 在四种触发组合下输出正确标签。
6. 周末、午间、盘中、收盘后的时段提示。

### 14.2 接口集成验证

保留一条手工验证命令：

```bash
curl -sS -L --max-time 15 \
  'https://fundmobapi.eastmoney.com/FundMNewApi/FundMNFInfo?pageIndex=1&pageSize=200&plat=Android&appType=ttjj&product=EFund&Version=1&deviceid=codex-verify&Fcodes=000001,110022,161725' \
  | jq '{ErrCode,Success,TotalCount,Expansion,Datas: [.Datas[] | {FCODE,SHORTNAME,PDATE,NAV,NAVCHGRT,GSZ,GSZZL,GZTIME}]}'
```

验收前需要至少记录一次 `GSZ` 非空样例。如果接口持续返回 `GSZ=null`，应暂停实现核心估值展示或由产品侧确认是否接受降级效果。

### 14.3 UI 验收

1. 默认进入确认净值模式。
2. 切换实时估值后，不修改配置内容。
3. 估值可用时，总览和持仓明细使用估值价格计算。
4. 估值不可用时，单只基金回退确认净值，并显示来源。
5. 批量接口失败时，页面仍能展示确认净值口径。
6. `估值触发` 和 `估值回落` 标签只在实时估值模式展示。
7. 回测页不出现实时估值入口或估值字段。
8. Popup 在常规宽度下文案不重叠，明细卡片信息不溢出。
